# Road to SaaS — hosted demo, Snowflake Native App, public signup

This doc sequences the three hosting phases grounded in what ships on `main`
today. It is the operator-facing companion to
[`docs/HOSTED_POC.md`](HOSTED_POC.md) (VM runbook) and
[`docs/snowflake-native-app/PUBLISH_CHECKLIST.md`](snowflake-native-app/PUBLISH_CHECKLIST.md)
(Native App publish gaps).

**Product boundary:** a managed public SaaS control plane is roadmap work gated
on pilot traction — see [`docs/PRODUCT_BOUNDARIES.md`](PRODUCT_BOUNDARIES.md).
Phase 0 and Phase 1 are real, shippable lanes today; Phase 2 is net-new.

---

## Phase 0 — Demo link (ops only, hours)

**Deliverable:** a live `https://demo.agent-bom.com` you invite testers to.

| What | Status on main |
|---|---|
| Platform compose | [`deploy/docker-compose.platform.yml`](../deploy/docker-compose.platform.yml) + [`deploy/docker-compose.hosted-poc.yml`](../deploy/docker-compose.hosted-poc.yml) |
| TLS front door | [`deploy/caddy/Caddyfile.hosted-poc`](../deploy/caddy/Caddyfile.hosted-poc) |
| Preflight gate | [`scripts/deploy/hosted_poc_preflight.py`](../scripts/deploy/hosted_poc_preflight.py) |
| One-shot bootstrap | [`scripts/deploy/demo-hosted.sh`](../scripts/deploy/demo-hosted.sh) |
| Post-TLS smoke | [`scripts/deploy/verify-hosted-demo.sh`](../scripts/deploy/verify-hosted-demo.sh) |
| Demo seed | `agent-bom quickstart --run --offline --force` (documented in HOSTED_POC) |

**What's left (no product code):**

1. VM + DNS A record → public IPv4
2. Inbound **443** only; SSH restricted to operator IPs
3. Four secrets + bootstrap admin key — generated by `demo-hosted.sh --generate-secrets`
4. `docker compose up` + Caddy TLS + quickstart seed
5. Mint a read-only tester key — `demo-hosted.sh --mint-tester-key`

**Unlocks:** real prospect demos of CSPM + vuln + compliance + graph today; first
feedback loop.

**Caveat:** invited testers only (operator-minted keys). No public signup.

### Quick start

```bash
# On the VM after DNS points at the host:
git clone https://github.com/msaad00/agent-bom.git && cd agent-bom
scripts/deploy/demo-hosted.sh \
  --generate-secrets \
  --domain demo.agent-bom.com \
  --acme-email you@example.com \
  --seed \
  --mint-tester-key \
  --wait-health
scripts/deploy/demo-hosted.sh --print-caddy --domain demo.agent-bom.com
# install Caddy, then:
scripts/deploy/verify-hosted-demo.sh --domain demo.agent-bom.com
```

## Phase 1 — Snowflake Native App (weeks, best-fit first product)

**Deliverable:** customers install agent-bom inside their own Snowflake account
(SPCS); data never leaves.

| What | Status on main |
|---|---|
| Native App package | [`deploy/snowflake/native-app/`](../deploy/snowflake/native-app/) |
| Install runbook | [`docs/snowflake-native-app/INSTALL.md`](snowflake-native-app/INSTALL.md) |
| Provider lane | [`docs/snowflake-native-app/MARKETPLACE.md`](snowflake-native-app/MARKETPLACE.md) |
| Dry-run packaging | [`.github/workflows/release-snowflake.yml`](../.github/workflows/release-snowflake.yml) |
| Contract tests | [`tests/test_snowflake_native_app.py`](../tests/test_snowflake_native_app.py) |

**Already built (~80%):** manifest, SPCS service specs (API/UI + opt-in scanner/MCP),
setup SQL, customer grants template, network policies, keypair auth template,
Snowpark Dockerfile, Streamlit dashboard (alternate path), release workflow
(validates package, pins image tags).

**What's left:** see the numbered checklist in
[`docs/snowflake-native-app/PUBLISH_CHECKLIST.md`](snowflake-native-app/PUBLISH_CHECKLIST.md).

**Unlocks:** enterprise distribution with the strongest trust story (in-account, no
data egress) — sidesteps most Phase-2 SaaS hardening and the DSPM “we don't read
your data” objection.

## Phase 2 — Managed SaaS / public signup (net-new, traction-gated)

**Deliverable:** anyone clicks the link, signs up, pays, uses it.

| What | Status on main |
|---|---|
| Multi-tenant scoping | Hardened tenant isolation in API stores |
| Per-tenant quotas | `postgres_tenant_quota`, rate-limit middleware |
| Enterprise identity | SCIM, RBAC, four auth modes |
| Connection broker | Encrypted cloud credential storage |

**P0 — cannot launch without:**

1. Self-serve signup/identity — public registration + social/SSO sign-up (no `/signup` today)
2. Tenant auto-provisioning on signup + hard per-tenant isolation (per-tenant encryption keys)
3. Public-internet abuse hardening — WAF/bot/DoS on top of existing quotas

**P1 — to charge:**

4. Billing/metering — plans, Stripe, usage→invoice (no Stripe integration today)
5. Vendor compliance — SOC2/ISO for the hosted service, DPA, sub-processors, pen test

**Unlocks:** true self-serve growth — weeks of net-new engineering; as a security
vendor, #5 is table stakes.

## Recommended sequence

```text
Now ──► Phase 0 demo link (hours, ops)
          │
          ├─► Phase 1 Snowflake Native App publish (weeks)  ◄── do this next
          │      best trust story, ~80% done, enterprise lane
          │
          └─► Phase 2 public SaaS (weeks+, traction-gated)
                 only after pilots prove demand
```

**Take:** ship Phase 0 immediately (it's free ops work), make Phase 1 the real
near-term product milestone, and gate Phase 2 behind pilot signal — exactly as the
editions model intends.

**Orthogonal (detection depth, not hosting):** DSPM content-classification engine
and CSPM real-time/Azure-IAM improve the product regardless of hosting phase.
