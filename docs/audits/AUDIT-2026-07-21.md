## §B. What changed in two weeks (persona-relevant)

### Release ladder

`0.93.5` → `0.94.x` → `0.95.0` → `0.96.0–0.96.4` → `0.97.0–0.97.1`

### Themes that moved persona scores

1. **Coverage lenses over one evidence layer** — CSPM · ASPM · DSPM · AISPM · vuln mgmt as overlapping disciplines (`finding_scope`), not siloed products (`0.96.0`).
2. **Identity + infra posture became real** — fail-closed CIEM authz evidence (AWS/Azure/GCP), KSPM live-cluster posture (kind-proven), opt-in redacted DSPM (DB + Azure Blob).
3. **Attack paths at estate scale, honestly bounded** — partitioned campaign engine with explicit caps and `LIMITED` (never false `COMPLETE`); MITRE kill-chain UI.
4. **Compliance honesty + provenance** — zero-evidence ≠ pass; own-worded CIS titles; NIST 800-53 Rev 5 catalog with OSCAL provenance + ISO crosswalk; framework library lock-in across CLI/API/UI/MCP.
5. **Operator cockpit + IA** — premium both-theme dense UI; Connections hub; Asset Inventory; retention + labeled ~90d windows; human↔agent parity settings.
6. **Auth / audit trust** — single-sourced auth posture; principal-bound API keys; tenant-scoped audit chains; JWT-invalid fail-closed; sealed-secret self-posture.
7. **Deploy / drift proofing** — collector sidecar image; Helm self-consistency; version-alignment CI gate across deploy/docs surfaces; Alembic as schema authority.
8. **Runtime clarity** — published fail-open/fail-closed matrix; bounded gateway relays; observe→enforce propose-only bridge; skill-scanner prohibition FP fixed.

## §C. Persona scorecards

### C1. AppSec / GRC — **8.5** (was ~7)

**Value proofs**

- Honesty enforced in scoring code (missing evidence → `not_evaluated`; no double-count of subsumed findings).
- Provenance-pinned NIST catalog + CI integrity gate.
- One evidence model → SARIF / CycloneDX 1.7 / SPDX / OpenVEX round-trip / CSV (formula-safe) / Parquet / OCSF.

**Gaps**

1. Non-NIST framework crosswalks remain vendor-asserted (need assessor-grade methodology docs).
2. Default local compliance bundle is unsigned unless HMAC configured.
3. Audit-period workflow (reviewer sign-off per control) still thin; hub-findings UI remains API-oriented by design.

**Verdict:** Compliance story moved from broad-but-occasionally-flattering to provenance-pinned and honest-by-construction.

### C2. Platform / SRE — **8.5** (was ~7)

**Value proofs**

- Helm profile matrix lint/template green; production-shaped chart (migration Job fail-closed, PDBs, HPA, ExternalSecrets, ServiceMonitor).
- Auth posture testable + self-reporting (`/v1/self-posture`); sealed vs literal secrets.
- Labeled retention windows; KSPM CronJob least-privilege ClusterRole; collector image separates cloud SDK churn.

**Gaps**

1. Some example values files fail standalone `helm template` without a Postgres base (day-0 trap).
2. Scheduled image/KSPM CronJobs write to `emptyDir` unless `--push-url` wired — silent evidence loss.
3. Postgres story is logical-dump / BYO-HA; migration cadence is high (runbook expectation).

**Verdict:** From capable chart with soft edges → self-auditing, drift-gated, fail-closed control plane an SRE can defend in review.

### C3. Agent builders — **8.5** (was ~7.5–8)

**Value proofs**

- Live MCP instantiation matches README (76/6/8) and CI gate.
- Proxy allow/block + AES-CMAC hash-chained audit verified; `/healthz` publishes fail-mode matrix.
- Skill scanner no longer flags legitimate `## Never` prohibition language (#4050).

**Gaps**

1. Signed MCP scan attestations: foundation merged / epic closed, but **no user-reachable CLI/API/MCP surface** yet — changelog “Added” over-implies reachability.
2. `audit` CLI exit-code contract inconsistent without `--json` when session summary missing.
3. Observe→enforce proposals lack a first-class propose/review/import workflow; gateway is HTTP-upstream oriented.

**Verdict:** Runtime enforcement graduated from claimed to proven; attestation story needs a front door.

### C4. Product manager — **8.5** (was 7.5)

**Proofs:** `scan --demo --offline` fast path; count + version-alignment gates; empty states stay truthful (`no_data`, N/A until evidence).
**Gaps:** Public demo host **HTTP 502** (reconfirmed this audit); Parquet-without-extra hint can swallow `[lake]` in rich console; demo parquet path exits partial.

### C5. Sales engineer — **8.5** (was 7)

**Proofs:** CLI → findings push → campaigns/overview; MCP tools/list; gateway `init-policy`; Helm templates for default / BYO-Postgres / EKS-IRSA; connect wizard + StackSet path.
**Gaps:** Demo host down; findings push alone does not populate graph (“run a scan first”); API local path scans disabled-by-default (correct, but demo friction).

### C6. Account executive — **7.5** (was 6.5)

**Proofs:** Three lanes consistent across README/docs/Helm; coverage lenses map to budget vocabulary; `EDITIONS.md` / entitlements honest (`lane: oss`); self-hosted governance story coherent.
**Gaps:** No GA managed hosted offering from this repo (adoption narrative, not revenue); dead demo URL; breadth risks message dilution.

### C7. Security engineer — **8.5** (was 7)

**Proofs:** Canonical finding envelope; CIEM fail-closed + evidence-health; KSPM/DSPM first-class; partitioned attack-path campaigns; auth 401/403/200 contract matrix.
**Gaps:** Store-backed UnifiedGraph proven but default-unwired; Azure/GCP CIEM thinner field mileage than AWS; DSPM opt-in/sampling must stay in every claim; access-review/JIT surfaces less live-proved in this window.
